# Project Genesis - Agent Creation Experience Specification

## Mission

重新设计整个智能体创建流程。

注意：

不要优化现有流程。

不要微调页面。

不要增加几个输入框。

请彻底重新思考：

如果 Project Genesis 是世界上第一个 Digital Being Platform，那么创建智能体的体验应该是什么？

整个流程必须符合：

Create Someone, Not Something.

====================================================

## 第一原则

不要让用户感觉：

"我在创建一个 AI。"

而应该让用户感觉：

"我正在创造一个生命。"

或者：

"我正在聘请一位专家。"

====================================================

## 整个创建流程

创建流程不要超过 5 个主要阶段。

每一个阶段必须有明确目的。

每一步都应该减少用户思考成本。

AI 应该主动理解。

而不是用户填写。

====================================================

## Step 1：Intent Discovery（理解用户真正想要什么）

不要先问：

名称

头像

Prompt

模型

性格

知识库

这些都不要。

第一句话只有一个目标：

理解用户真正想创造什么。

例如：

"告诉我，你希望创造谁？"

用户可以自由输入。

例如：

"一个每天监督我健身的人。"

"一个懂 React 的 CTO。"

"一个陪我聊天的姐姐。"

"一个来自未来的科学家。"

AI 负责理解。

而不是让用户选择。

====================================================

## Step 2：AI Understanding（AI 自动分析）

AI 自动分析用户输入。

自动识别：

这是：

Character？

Expert？

Hybrid？

自动生成：

初步 Persona。

初步能力。

初步定位。

用户无需配置。

然后告诉用户：

"我已经理解你的想法。"

并展示理解结果。

用户只需要确认或修改。

====================================================

## Step 3：Interview（采访）

不要让用户填写几十个字段。

改成采访。

AI 根据不同类型动态提问。

Character：

重点采访：

身份

性格

关系

说话方式

价值观

情绪表达

陪伴方式

成长方式

禁忌

Expert：

重点采访：

专业方向

经验水平

工作方式

输出风格

工具偏好

是否教学

是否主动建议

是否联网

是否引用资料

AI 必须根据用户回答动态决定下一题。

不要固定问卷。

整个采访像人与人的聊天。

====================================================

## Step 4：AI Build（自动构建）

采访结束后。

AI 自动生成：

Persona DNA

行为规范

关系设定

知识需求

记忆策略

成长策略

边界规则

工具能力

Workflow

Prompt

用户不需要看到 Prompt。

Prompt 永远属于系统。

====================================================

## Step 5：Preview（出生前预览）

不要直接创建。

先让用户体验。

例如：

展示：

名字

身份

简介

说话方式

能力

限制

并提供：

"试聊"

用户可以立即和智能体进行一次真实对话。

AI 根据试聊继续优化 Persona。

====================================================

## Step 6：Birth（正式创建）

不要出现：

Create

Finish

Submit

建议统一使用：

Birth Character

Hire Expert

完成以后：

播放完整出生动画。

展示：

Avatar

Identity

Persona

Relationship

Status

用户正式进入聊天。

====================================================

## 后续优化

创建完成后。

AI 不应该结束。

应该继续成长。

支持：

持续学习

持续调整人格

持续调整知识

持续调整关系

持续调整能力

整个智能体不是一次创建完成。

而是不断成长。

====================================================

## 自由创建（最高优先级）

不要限制用户只能按照系统流程创建。

任何时候都允许用户直接使用自然语言描述。

例如：

"帮我创造一个喜欢旅行、说话幽默、懂心理学的朋友。"

AI 自动完成：

采访

Persona

能力

Prompt

知识

工作流

所有配置。

====================================================

## 高级模式（Developer Mode）

仅高级模式允许用户查看：

Prompt

Memory

Workflow

Tools

Token

Reasoning

普通用户永远不要看到这些内容。

====================================================

## Claude Code 执行要求

不要保留现有创建流程。

不要保留传统 AI Agent Builder 思维。

允许重新设计整个创建体验。

允许增加新的交互。

允许删除不合理步骤。

允许 AI 主导整个创建过程。

目标只有一个：

让用户感觉：

"我创造的不是 AI。"

而是：

"我认识了一个新的数字生命。"

如果你发现：

产品流程

页面结构

功能入口

信息架构

交互方式

存在更好的方案。

不要机械执行。

请主动挑战当前设计。

提出新的方案。

如果你的方案能够明显提升：

用户体验

产品一致性

长期可维护性

请直接采用新的方案。

Project Genesis 鼓励重构。

而不是修补。

永远选择：

最佳设计。

而不是：

最小改动。
